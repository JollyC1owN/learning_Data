模块化

1、模块化简介

1.概念

将一个复杂的程序依据一定的规则(规范)封装成几个块(文件), 并进行组合在一起

2.模块化的好处

避免命名冲突(减少命名空间污染)
更好的分离, 按需加载
更高复用性
高可维护性

3.页面引入加载很多个script的问题

请求过多
依赖模糊
难以维护

4.没有模块化时

在没有出现模块化时，用的是IIFE增强模式
即：匿名函数自调用，暴露出一些东西

/**

 * IIFE模式增强 : 引入依赖

 * 这就是现代模块实现的基石

 */

((window, $)=> {

  //数据

  let data = 'atguigu.com'



  //操作数据的函数

  function foo() { //用于暴露有函数

    console.log(`foo() ${data}`)

    $('body').css('background', 'skyblue')

  }



  function bar() {//用于暴露有函数

    console.log(`bar() ${data}`)

    otherFun() //内部调用

  }



  function otherFun() { //内部私有的函数

    console.log('otherFun()')

  }



  //暴露行为

  window.myModule = {foo, bar}

})(window, jQuery)

2、commonJS

1、存在的关系

在CommonJs模块化中，存在这样一个关系
module.exports = exports = { }
当一个模块中同时使用了两种，以module.exports为准—（避免这样的情况出现）
当一个模块中使用了多个module.exports；以最后一次为准
当一个模块中使用了多个exports.xxx；则是向同一个对象中追加键值对;对象指向没变

2、语法
暴露：

module.exports = value
exports.xxx = value

module.exports = {

  name:'kobe',

  age:18,

  setName(name){

    this.name = name

  }

}

-------------------------------------

exports.haha = [1,2,3,4,5]

exports.haha2 = {m:1,n:2}

引入

require(xxx)
第三方模块:xxx为模块名
自定义模块: xxx为模块文件路径 ./xxxx

let module1 = require('./modules/module1')

let module2 = require('./modules/module2')

let module3 = require('./modules/module3')

let uniq = require('uniq')

3、浏览器端与服务端

服务端：

  • 在node环境下运行app.js的两种方法(任选其一)：

    • 第一种方法：用命令启动: node app.js

    • 第二种方法：用工具启动: 右键 –> Run ‘xxxxx.js’

浏览器端：

  • 全局安装browserify，命令: npm install browserify -g（若此步骤报错，请使用管理员身份打开webstorm，再次执行即可）

  • 第二步，输入命令browserify js/src/app.js -o js/dist/bundle.js
    -页面使用引入
    <script type="text/javascript" src="js/dist/bundle.js"></script>

3、ES6（重中之重）

1、语法
暴露模块-export:

分别暴露：export 暴露内容

//分别暴露

export function foo() {

  console.log('module1 foo()');

}

export function bar() {

  console.log('module1 bar()');

}

export const DATA_ARR = [1, 3, 5, 1]

统一暴露：export {暴露内容1, 暴露内容2}

//统一暴露

let data = 'module2 data'

function fun1() {

  console.log('module2 fun1() ' + data);

}

function fun2() {

  console.log('module2 fun2() ' + data);

}

export {fun1, fun2}

默认暴露：export defalut 暴露内容

//默认暴露

export default {

  name: 'Tom',

  setName: function (name) {

    this.name = name

  }

}

引入模块-import:

import {暴露内容1, 暴露内容2} from ‘./module1’
import module3 from ‘./module3’ —-默认暴露的引入方式

//引入【分别暴露】的模块

import {demo,test} from './module1'

//import * as module1 from './module1' //这种写法会将所有module1里暴露内容收集到一个对象里



//引入【统一暴露】的模块

import {haha1, haha2, haha3} from "./module2";

//import * as module2 from './module2'



//引入【默认暴露】的模块

import module3 from './module3'

//import * as module3 from './module3'



//引入module4，module4即使用了默认暴露，也使用了分别暴露

import module4,{fun,fun2} from './module4'



//引入第三方模块

import $ from 'jquery'

2、准备工作
安装babel-cli, babel-preset-es2015和browserify

第一步，全局安装：npm install babel-cli browserify -g
备注：若全局已经安装过browserify，可以在第一步中去除browserify

定义.babelrc文件(给babel指定具体的任务)，内容如下

{

   "presets": ["es2015"]

}

编译源代码

  • 第一步：使用Babel将ES6编译为ES5代码，操作后Babel将es6的模块化语法，转换成了CommonJS模块化语法（浏览器不识别）
    命令为: babel js/src -d js/lib

  • 第二步：使用Browserify编译js上一步生成的js
    命令为: browserify js/lib/app.js -o js/lib/bundle.js

  • 备注：第一步操作后Babel将es6的模块化语法，转换成了CommonJS模块化语法（浏览器不识别），所以需要第二步用Browserify再次编译。

页面中引入测试

<script type="text/javascript" src="js/lib/bundle.js"></script>

6、node和npm

1、 Node是什么？

是一个js运行环境

2、node与npm是什么关系？

node想帮助全球开发者管理优秀的项目，创造了一个优秀的包管理器，叫做npm。
npm：全称：Node package manager，安装了Node就会自动安装npm
包（库、项目）：电脑上的一个普通文件夹，具备了一些特点（包含了package.json）就变成了包，命令：npm init

查看自己node的版本命令：node -v
查看自己npm的版本命令：npm -v

3、要求：

node版本8以上(不包含8版本）
npm版本5以上(不包含5版本)
为了让你的npm下载东西更迅速，请执行以下命令：
npm config set registry http://registry.npm.taobao.org/

4、包名的要求：

不能有中文
不能有大写字母
不能npm上已经存在的包重名

5、常见命令

安装一个别人的包： npm install xxxx
